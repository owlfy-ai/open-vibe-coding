# Security Policy

## Reporting Security Issues

Please report security issues through GitHub issues, but do not publicly include sensitive information such as:

- API keys
- access tokens
- private prompts or generated project data
- personal user data
- full exploit details that could immediately harm users

Open an issue with a minimal, non-sensitive description and clearly mark it as security-related. The maintainers can then coordinate next steps in a safer channel if needed.

## Sensitive Data

Open Vibe Coding can handle user-provided API keys for AI, search, and image providers. Do not include real credentials in bug reports, screenshots, logs, or pull requests.

## Supported Versions

Security fixes are intended for the latest version on the main branch unless a separate release branch is explicitly maintained.
